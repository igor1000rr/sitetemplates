# Запрос доступов для запуска AITempl

Чтобы запустить магазин, нужны ключи доступа к нескольким сервисам. Ниже — что
прислать и где это взять. Всё сразу не обязательно: для старта продаж достаточно
**блока 1** (хранилище + оплата), остальное можно подключить позже.

> Почту мы берём на себя (Resend) — от вас она не требуется.

---

## 1. Обязательно для старта

### 📦 Хранилище файлов — Timeweb S3
Где: панель Timeweb (`timeweb.cloud`) → раздел **«Облачное хранилище S3»**.
Прислать:
- **Access Key ID** и **Secret Key** (раздел «Ключи доступа» — секрет показывается один раз);
- **точное имя бакета** (хранилища), где лежат/будут лежать файлы шаблонов.

### 💳 Приём оплаты — ЮKassa
Где: личный кабинет ЮKassa (`yookassa.ru`), магазин оформлен на ваше ИП.
Прислать:
- **shopId** (идентификатор магазина);
- **секретный ключ** (Настройки → Интеграция → выпустить ключ).

---

## 2. Желательно сразу после старта

### 🔔 Уведомления о заказах — Telegram
- **Токен бота**: в Telegram написать **@BotFather** → команда `/newbot` → он выдаст токен.
- **Ваш chat_id**: написать **@userinfobot** → пришлёт числовой ID (туда будут
  приходить уведомления о заказах). После этого напишите своему боту `/start`.

---

## 3. По желанию (можно подключить позже)

- **AI-чат и AI-подбор** — ключ **OpenAI** (`platform.openai.com` → API keys).
- **Вход через Google** — Client ID + Secret (`console.cloud.google.com` → Credentials →
  OAuth client, тип Web). Redirect URI: `https://aitempl.ru/api/auth/google/callback`
- **Вход через Яндекс** — Client ID + Secret (`oauth.yandex.ru` → новое приложение).
  Redirect URI: `https://aitempl.ru/api/auth/yandex/callback`
- **Аналитика посещений** — номер счётчика **Яндекс.Метрики** (`metrika.yandex.ru`)
  и/или **Measurement ID** Google Analytics вида `G-XXXXXXX` (`analytics.google.com`).

---

## Как прислать безопасно
Ключи — это доступ к деньгам и данным, поэтому:
- не отправляйте их открытым текстом в обычном мессенджере или почте;
- лучше использовать одноразовую ссылку (например, `onetimesecret.com`) или передать
  в защищённом канале.

Любой ключ при необходимости можно отозвать и выпустить заново.
